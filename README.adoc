An *embeddable, domain-agnostic multi-channel notification library* for Java / Spring Boot. The jar is
the product — there is no server to run.

== Why notify4j

* *Apprise/shoutrrr-style URL channel config, natively on the JVM* — declare channels as a list of URLs
  (`slack://…`, `webhook://…`, `pagerduty://…`); no Python/Go sidecar. (No native Java equivalent exists.)
* *Ops-grade alerting semantics* — status-transition filtering (notify on real changes only), runtime
  mute, and reminders for unresolved states.
* *Spring-free core + a thin starter* — use the engine anywhere, or drop in the Spring Boot starter and
  configure `notify4j.urls`.
* *Channels included* — Slack, MS Teams, Discord, Mattermost, Rocket.Chat, Google Chat, Telegram,
  generic webhook, ntfy, Gotify, Pushover, Pushbullet, Zulip, SMS (Twilio), Signal, WhatsApp,
  PagerDuty, OpsGenie, and email.

== Get started

Add the Spring Boot starter (pulls in `notify4j-core`):

[source,xml]
----
<dependency>
    <groupId>org.alexmond</groupId>
    <artifactId>notify4j-spring-boot-starter</artifactId>
    <version>1.0.0</version>
</dependency>
----

Declare a `NotificationAdapter` bean (teaches notify4j how to read your event type) and list
channels under `notify4j.urls`:

[source,yaml]
----
notify4j:
  urls:
    - slack://hooks.slack.com/services/T000/B000/XXXX
    - pagerduty://<routing-key>?tags=failed
----

The starter auto-configures a `Notifications` facade you inject and call: `notifications.send(event)`.
See the link:notify4j-sample[`notify4j-sample`] module and the
https://www.alexmond.org/notify4j/current/index.html[reference docs] for a full walkthrough.

API reference (javadoc.io):
https://javadoc.io/doc/org.alexmond/notify4j-core/latest/index.html[notify4j-core] ·
https://javadoc.io/doc/org.alexmond/notify4j-spring-boot-starter/latest/index.html[notify4j-spring-boot-starter].

== Modules

[cols="1,3"]
|===
| `notify4j-core` | Spring-free engine: `Notifier<E>` SPI, wrappers, URL parser, facade, channels.
| `notify4j-spring-boot-starter` | Auto-configuration: binds `notify4j.urls`; email optional.
| `notify4j-bom` | Dependency BOM — import to align versions and "pick all".
|===
